FAANG-Level System Design Insight

Why gRPC is used in FAANG:

Google → invented gRPC → internal RPC communication
Netflix → microservices communication
Uber → real-time dispatch system
Amazon → internal service mesh


Interview Questions (FAANG Level)
1. When would you choose gRPC over REST?

Answer:

Low latency required
Internal microservices
Streaming needed
Strong contract required
2. Why is gRPC faster?

Answer:

Uses HTTP/2 (multiplexing)
Binary serialization (protobuf)
Smaller payload size

3. What are drawbacks of gRPC?

Answer:

Hard to debug (binary)
Not browser-friendly
Learning curve (.proto)
4. How do you secure gRPC?

Answer:

TLS encryption
JWT metadata
API gateway (Envoy)
5. REST vs gRPC in Microservices?

Answer:

External → REST
Internal → gRPC
